# 設計的工作方式正在改變，但不是你想的那樣

> 層級：上 — Design in AI Era

## 設計師的工作流在 2026 年長什麼樣？

兩年前，一個產品從概念到上線的設計流程大概長這樣：PM 寫 spec，設計師在 Figma 做 mockup，幾輪 review 之後交給工程師，工程師照著設計稿刻 UI，來回修改直到跟 mockup 一致。這個流程在大公司可能要跑兩到四週，在小團隊裡也至少要幾天。

2026 年的流程正在變成另一個形狀。你可以把一個想法用自然語言描述給 AI，它會直接產出可以互動的 prototype。不是線框圖，不是靜態 mockup，而是你可以點進去操作、看到動畫和轉場的真實介面。如果你不喜歡配色，跟它說一句話就換掉。如果你想看另一種佈局風格，它可以同時產出三個方向讓你比較。

這不是科幻情節。Claude Design、v0、Galileo 這些工具已經在做這件事。但它們真正改變的東西，比「設計變快了」更深層。

## Pre-AI vs Post-AI：從 Figma 手稿到一次對話產出設計

傳統設計流程的核心是一系列的人工翻譯步驟。商業需求翻譯成 spec，spec 翻譯成 wireframe，wireframe 翻譯成 visual design，visual design 翻譯成 code。每一次翻譯都有資訊損耗，每一次 handoff 都有來回修改的成本。一個按鈕的圓角在 Figma 裡是 8px，到了工程師手上可能變成 6px，review 的時候再改回來。這類摩擦佔據了設計工程協作裡大量的時間，但沒有人會把它寫進 sprint planning。

AI 設計工具正在壓縮這些翻譯步驟。當你能用一句話描述你想要的介面，AI 直接產出 code-ready 的 prototype，wireframe 到 visual design 到 code 這三層之間的 handoff 就消失了。你不再需要一個人畫圖、另一個人照著圖寫 code，因為圖和 code 從一開始就是同一個東西。

但這並不代表設計師的角色消失了。它代表設計師的工作重心從「手動產出交付物」移向「判斷和決策」。以前你花 60% 的時間在 Figma 裡調 pixel，40% 的時間在想「這個流程對不對」。現在調 pixel 的工作被 AI 接走了，你 100% 的時間都應該花在判斷上：這個動線合不合理？這個文案會不會造成誤解？這個配色傳達的情緒對不對？

## AI 設計工具能取代設計師嗎？

短答案：不能。但它正在重新定義「設計師」這個角色的邊界。

一個有趣的現象是，AI 設計工具對兩種人最有用。第一種是有設計判斷力但不會寫 code 的設計師 — 他們現在可以直接把想法變成可以跑的 prototype，不用等工程師。第二種是會寫 code 但沒有設計訓練的工程師 — 他們現在可以拿到多個專業級的設計方向，從裡面挑一個來實作，而不是自己硬湊一個勉強能看的 UI。

AI 不擅長的事情恰恰是設計裡最重要的部分：理解使用者的真實情境、在多個看起來都不錯的方案裡判斷哪一個最合適、決定什麼東西不該放進畫面裡。這些能力來自經驗、來自對人的理解、來自品味。AI 可以一次給你十個方向，但挑出對的那一個是人的工作。

## 判斷力才是新的稀缺資源

當生產設計稿的成本趨近於零，設計的價值就完全集中在判斷上。這跟攝影的演變很像 — 當每個人手上都有一台能拍出技術上完美照片的手機，「會拍照」不再是稀缺能力，「知道什麼值得拍、怎麼拍才有意義」才是。

對於正在建產品的小團隊來說，這意味著一件實際的事：你不一定需要一個全職設計師來產出設計稿，但你絕對需要一個有設計判斷力的人來做決策。這個人可以是創辦人自己、可以是一個有設計敏感度的工程師、可以是一個兼職的設計顧問。關鍵不在於誰來畫圖，而在於誰來拍板「就是這個方向」。

工具的進步讓執行成本下降了，但判斷的門檻沒有變。在接下來的系列文章裡，我們會深入介紹幾個具體的 AI 設計工具 — 它們各自擅長什麼、適合什麼場景、以及怎麼把它們串進你的工作流裡。
