## 這個案例的起點：一個 2012 年的企業官網

一家 1987 年成立的精密鑄造廠，做脫蠟精密鑄造，客戶遍及汽機車、國防、氣動工具、閥門等產業。官網大概是 2012 年做的，XHTML + iframe + table 排版，SSL 過期，英文版和日文版寫著「建構中」十三年。

這是台灣中小企業的常態。不是不想改，是改的成本結構不對 — 找外包做一個企業官網動輒十幾萬，做完之後要更新內容還得再找人，或是學一套 CMS 但又沒有人會維護。於是網站就一直停在那個年代。

這篇記錄的是用 AI 工具幫這家工廠重做網站的完整過程。兩個版本，兩種不同的工具和方法，從研究到上線。

## 研究同業之後學到的 B2B 網站設計原則

動手之前先研究了 10 家國內外精密鑄造同業的網站。這個步驟不能跳過，因為 B2B 製造業網站有它自己的設計語言，跟 SaaS landing page 或電商網站是完全不同的邏輯。

歸納出的共同模式：

- **首頁結構照著客戶的決策路徑排**：Hero 建立第一印象 → 製造能力和認證資質 → 服務的產業別 → 報價入口。採購和工程師上你的網站只有一個目的：判斷「你能不能做、品質可不可靠」，所有設計決策都應該服務這個目標
- **深色沉穩配色**：深藍或深灰搭白底再加一個強調色，這在製造業裡幾乎是慣例
- **大量真實照片**：工廠實拍、設備實拍、產品實拍。stock photo 在 B2B 製造業的語境裡反而會降低信任感
- **導覽不超過六項**，CTA 統一是「Request a Quote」

台灣同業的共同問題則是技術債：table 排版、沒有響應式設計、沒有 SSL、SEO 幾乎為零。這不是個案，而是整個產業帶的現況。

這個研究階段最重要的發現是：**做得好的不是最華麗的，而是讓工程師和採購三秒內找到答案的。** 這個原則決定了後面所有的設計方向。

## 用 Claude Code 手刻第一版

### 盤點現有素材

把舊網站 30 幾個 HTML 檔全部爬下來之後發現，實際有價值的內容很少：公司沿革五個年份、品質承諾三句話、八類產品各一張照片、五台檢驗設備各一張照片、一張製程流程圖、聯絡資訊。這些東西一頁式就能全部放完，舊網站拆成 30 幾頁純粹是 iframe 時代的技術限制，不是因為內容真的多到需要分頁。

### 配色決策

配色從品牌既有的視覺資產出發，而不是從零發明。這家公司的 logo 是紅藍黃三原色，但 25 週年紀念視覺用的是紫藍色調，顯示品牌其實一直偏好紫色。

最後決定走紫 + 金黃的組合。理由有三：紫色在精密鑄造同業裡沒人用，辨識度天然就高；紫代表精緻和技術感、金代表金屬和品質感，語意上跟精密鑄造對得上；紫和金是互補色，對比強但不吵。

### 技術選擇和實作

用 Claude Code 直接寫純 HTML + CSS + vanilla JS，零框架依賴，整個網站就是一個 `index.html`。選擇零依賴不是技術偏好，是因為這個網站的需求極簡 — 靜態內容、不需要後端、不需要狀態管理 — 加任何框架都是多餘的複雜度。

頁面結構跟研究階段歸納的 B2B 模式一致：

```
Hero → 數據條 → 關於（時間軸 + 品質承諾）→ 產品 → 製程 → 設備 → CTA → 聯絡 → Footer
```

CSS 使用 OKLCH 色彩空間定義整套色彩系統，搭配 clamp() 做流體排版，讓字型大小和間距在不同螢幕尺寸之間平滑縮放而不是靠 breakpoint 跳階。動畫用 Intersection Observer 做 scroll-reveal，加上 staggered delay 讓多個元素依序出現。

部署在 GitHub Pages 上，當天就上線了。

### 踩到的坑

**GitHub Pages + Jekyll 的隱藏行為**：GitHub Pages 預設會跑 Jekyll 建置，而 Jekyll 會忽略底線開頭的檔案。從舊網站抓下來的產品照片有些是相機原始命名（像 `_MG_0062.jpg`），部署後全部 404。在根目錄放一個 `.nojekyll` 空檔案就解決了，但如果你不知道這個機制，debug 會花不少時間。

**素材品質的限制**：舊網站的產品照是「一堆零件散落在白底上」的合照，放進現代的卡片式佈局裡顯得格格不入。這個問題沒辦法靠 CSS 解決，最終還是需要重新拍攝。但先用現有素材上線、之後再迭代，比等到完美素材到齊才開始做要實際得多。

**表單設計的取捨**：精密鑄造的詢價不需要即時處理，也不需要複雜的表單欄位。一個 `mailto:` 連結預填好主旨和內容模板，客戶點了直接開 email app，比一個沒有後端支撐的假表單誠實得多。

## 用 Claude Design 重新設計第二版

第一版上線之後，用 Claude Design 對同一個案例做了第二次設計嘗試。Claude Design 是 Anthropic 推出的獨立設計工具，跟 Claude 聊天裡的 Artifacts 不同 — 它有持久的 canvas，可以反覆迭代累積設計資產，最後輸出一個完整的 handoff bundle 讓 Claude Code 等 coding agent 直接實作。

做法是把舊網站的網址、現有的照片素材和第一版的程式碼一起丟給 Claude Design，讓它重新設計。

### Claude Design 產出了什麼

Claude Design 產出了三個完整的設計方向，每個方向都是可以在瀏覽器裡直接跑起來的 React prototype。

以 Direction A「Clean Industrial」為例，走的是白底、大量留白、大字型的路線，帶有一套完整的 CSS 變數系統。它從 logo 的紅藍黃三原色衍生出四組配色方案 — 品牌藍、鍛橘、深紅、石墨灰 — 每一組都包含主色、輔助色、背景色和文字色的完整定義，透過 Tweaks Panel 可以即時切換。

字型方面它準備了三種組合：Inter Tight（現代無襯線）、Sora（幾何無襯線）、Source Serif 4（帶編輯感的襯線字型）。Hero 區塊有三種文案變體，針對不同的市場切入角度。每個模組都可以獨立開關。

更有意思的是它額外做了第一版沒涵蓋的頁面：產品詳情頁（product-page.jsx）和 RFQ 報價表單（rfq-form.jsx）。這些在 B2B 網站裡其實是核心功能，第一版因為時間限制跳過了，Claude Design 自己判斷這些應該要有。

### Handoff bundle 的結構

產出的 handoff bundle 長這樣：

```
project/
├── Yea Shinn Site A.html     ← 設計方向 A（進入點）
├── Yea Shinn Site.html       ← 其他設計方向
├── src/
│   ├── shared.jsx             ← 共用元件
│   ├── direction-a.jsx        ← 方向 A 完整實作（27KB）
│   ├── direction-b.jsx        ← 方向 B
│   ├── direction-c.jsx        ← 方向 C
│   ├── product-page.jsx       ← 產品詳情頁
│   ├── rfq-form.jsx           ← 報價表單
│   └── logo-paths.js          ← SVG logo 路徑資料
├── images/                    ← 所有圖片資產
└── README.md                  ← 給 coding agent 的實作指示
```

這個 bundle 是設計來被 Claude Code 消費的。README 裡明確指示：讀主設計檔，順著 import 讀完所有元件，理解整體結構後用適合的技術棧重新實作。prototype 的 code 不是 production code（用的是 CDN React + Babel 即時編譯），agent 的任務是匹配視覺產出，而不是複製內部結構。

## 兩個版本的差異：什麼時候該手刻、什麼時候該讓 AI 提案

| | V1：Claude Code 手刻 | V2：Claude Design |
|---|---|---|
| 投入時間 | 一個晚上 | 一次對話 |
| 設計決策 | 自己研究同業後判斷 | AI 提供三個方向，人來挑 |
| 配色來源 | 從品牌資產手動提煉 | AI 從 logo 衍生出四組方案 |
| 技術產出 | 純 HTML 單檔，可直接部署 | React prototype，需要轉換 |
| 涵蓋範圍 | 單頁首頁 | 首頁 + 產品頁 + 報價表單 |
| 後續成本 | 改 code 就是改設計 | 需要 Claude Code 再轉一次 |

兩個版本代表的是兩種工作模式。V1 是「我知道我要什麼，AI 幫我寫」— 你已經做完研究、定好方向，Claude Code 就是你的打字員，效率最高。V2 是「我想看看還有什麼可能性，AI 給我選項」— 你還在探索階段，或者你想看看自己的判斷跟 AI 的提案差多少。

兩者不是互斥的，而是設計流程裡不同階段的最佳工具。如果你對設計方向很有把握，直接用 Claude Code 手刻會更快。如果你不確定、或者你想挑戰自己的預設，先讓 Claude Design 給你選項，再用 Claude Code 把選定的方向實作出來。

這個案例還在進行中，接下來的步驟是從三個方向中定案、用 Claude Code 轉成 production HTML、設定自訂域名並部署。
